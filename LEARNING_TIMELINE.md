# Learning Timeline

A living timeline for a developer who is learning deeply and building publicly.

---

## 1. Curiosity phase

- Started with curiosity and limited tools.
- Learned by observing apps, experimenting with configuration, and asking why systems behaved a certain way.
- Early focus: becoming comfortable with technology without waiting for formal resources.

---

## 2. Independence phase

- Bought the first personal device and shifted into daily experimentation.
- Built early automation experiments, small scripts, and simple prototypes.
- This was the start of intentional engineering practice.

---

## 3. Resilience phase

- Grew a content and community presence, then rebuilt after platform setbacks.
- Learned the importance of backups, moderation, and living systems instead of perfect launches.
- The biggest lesson: systems need recovery paths as much as new features.

---

## 4. Engineering phase

- Moved from surface-level tricks to deeper automation, API work, and reliability.
- Built Telegram + GitHub logging flows, workflow automations, and bot integrations.
- Started treating every experiment as a system with inputs, outputs, and failure cases.

---

## 5. Current lab

- Active projects now include AI agents, developer tools, workflow automation, and hardware experiments.
- Focus on real usage: telemetry, observability, and practical feedback.
- This repo is now a lab for architecture notes, build logs, and system playbooks.

---

## 6. Maintenance & reasoning phase

- Shifted from building solo projects to maintaining open source (Birthday Bloom, Ghost Engine v5).
- Deepened AI work: directing agents to build workflows instead of just testing prompts.
- Started a new track in cybersecurity.
- Dedicated real time to physics, maths, and critical thinking to build stronger mental models before writing code.

---

## 7. Forward path

- Explore device communication, sensor data, and hardware-software bridges.
- Keep learning how to direct AI effectively and secure systems.
- Share what works, what breaks, and what is learning-worthy.

---

## Guiding cycle

- Learn
- Test
- Build
- Repeat

This timeline is not a finished story. It is a record of practice and an invitation to keep building.
